# 站点维护注意事项

这份文档记录本轮改版讨论中确定的产品、内容和页面结构注意事项。后续新增页面、文章或重构模板时，优先按这里检查。

## 导航与信息架构

- 全站主导航保持统一：`首页 / 文章 / 专题 / 项目 / 关于`。
- 小屏导航采用紧凑横向滚动，新增导航项时要检查 360px 宽度下是否仍能顺畅访问。
- 不再单独保留“分类”页。内容主题用标签页承载，系列阅读路径用专题页承载。
- “方法”和“关于”已经合并。研究方法、数据来源、AI 使用边界和风险声明统一放在 `/about/#method`。
- RSS、搜索、反馈属于工具入口，不放进主导航内容结构里。
- 页脚只放低频入口，例如 RSS、搜索、研究方法和联系，不承担主要导航功能。

## RSS、SEO 与分享

- RSS 地址保持为 `/rss.xml`。
- RSS XML 需要保留 XSL 样式表引用，让浏览器打开时显示成可读页面。
- RSS 入口应同时支持“打开 RSS”和“复制订阅地址”，并告诉用户复制后可以粘贴到 RSS 阅读器。
- canonical 和站点域名统一使用 `https://cphxnotes.com`。
- 页面需要保留 RSS alternate link、canonical、Open Graph 和 Twitter card 基础信息。
- 全站模板需要保留 `WebSite` JSON-LD。文章页需要保留 `BlogPosting` 和 `BreadcrumbList` JSON-LD。
- 默认 OG 图保留在 `public/og-default.svg`。
- 文章可以通过 frontmatter 的 `ogImage` 或 `cover` 覆盖默认分享图，路径应指向 `public` 下的站内资源。
- 未配置 `ogImage` 或 `cover` 时，文章页自动使用 `/og/<slug>.svg` 动态生成的 SVG 分享图。
- 文章如有外部权威原文或迁移来源，可以使用 `canonical` 覆盖默认 canonical。

## 文章页体验

- 文章页保留自动目录、阅读进度条、上一篇/下一篇、同专题入口、本文要点和评论区。
- 文章页侧栏目录会根据滚动位置高亮，目录链接需要继续使用页内 hash。
- 文章页展示预计阅读时长，估算逻辑放在 `src/lib/posts.ts`。
- 文章页底部显示基于共同标签匹配的相关阅读，数量保持克制。
- 正文图片由全站脚本补充懒加载和异步解码属性。新增图片仍然要在 Markdown 里保留清晰 alt 文案。
- “本文要点”不要重复文章摘要或导语。摘要已经显示在标题下方，要点应来自正文结构或关键结论。
- 如果正文没有足够的二级标题，“本文要点”需要显示短空状态，不渲染空列表。
- 页面保持整体滚动，不做左右两栏分别滚动。
- 右侧栏不应被首页或文章内容高度拉伸。使用顶部对齐，让侧栏按自身内容高度显示。
- 首页右侧栏宽度、字号、行高、链接样式要和其他页面保持一致。
- 目录只展示正文二级标题。脚注自动生成的 `Footnotes` 不进入目录。
- 目录数量不要靠代码硬截断。文章本身要合并相近小节，正文目录建议少于 10 个。

## 内容结构

- 系列文章标题可以保留编号，但要带完整关键词，便于搜索结果和新读者判断内容。
- “从零看懂股票”系列文章要保持上一篇/下一篇的系列顺序。
- 新增或整理专题文章时，优先使用 `series` 和 `seriesOrder` 字段；旧文章仍可暂时用标签兜底。
- 文章如有较重要更新，可以填写 `updatedDate` 和 `updatedReason`，标题区会展示更新说明。
- 提交文章前运行 `npm run check:content`，优先解决 error，再评估 warning。
- 长文小标题不要过碎。主问题用二级标题，解释性细节用三级标题。
- 避免高频使用“不是……而是……”句式。需要转折时，优先用更直接的表达。
- 标签和专题概念要区分清楚：标签用于主题索引，专题用于顺序阅读路径。

## 搜索与归档

- 搜索页使用前端轻量索引 `/search.json`，搜索范围包括标题、摘要和标签。
- 搜索结果必须使用 DOM API 渲染，避免把文章标题、摘要或标签直接拼接成 HTML 字符串。
- 搜索页需要保持 `?q=` 查询同步，方便分享搜索结果。
- 搜索页保留标签快捷入口，标签按钮只负责填入搜索词，不替代标签归档页。
- 文章归档页按年份分组，保留总文章数和年度文章数提示。

## 项目与数据页

- 项目页数据统一维护在 `src/data/projects.ts`，页面只负责渲染。
- 新增项目时优先补齐 `eyebrow`、`title`、`description`、`status`、`meta` 和 `links`。
- 持仓页表格保留横向滚动，小屏必须显示滑动提示。
- 恐慌指数图表在小屏下允许横向查看，避免压缩到不可读。

## 新增文章检查清单

- frontmatter 包含 `title`、`description`、`pubDate`、`tags`、`draft`。
- 如果属于系列，添加对应 `series` 和 `seriesOrder`，并保留必要标签。
- 标题既保留系列感，也包含核心搜索关键词。
- 二级标题数量控制在 10 个以内，靠合并内容结构实现。
- 图片放在 `public/images/posts/<slug>/`，路径在 Markdown 中使用 `/images/posts/<slug>/<file>`。
- 图片文件名必须语义化，使用小写英文 kebab-case，例如 `gross-margin-calculation.svg`。不要保留 `ChatGPT Image...`、时间戳、空格、括号这类生成器默认文件名。
- 同一篇文章的配图统一放在文章 slug 目录下，不要直接散放在 `public/images/posts/` 根目录。
- 文章末尾保留必要风险提示，投资相关内容不构成投资建议。
- 提交前运行 `npm run check:content`。
- 提交前运行 `npm run build`。
