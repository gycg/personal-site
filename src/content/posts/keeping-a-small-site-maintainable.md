小网站最容易出现的问题，不是功能不够，而是很快堆出一套超出需求的复杂结构。

## 少依赖

如果一个功能可以用平台能力或少量原生代码解决，就不必急着安装库。依赖越少，升级、排错和迁移越简单。

这个博客没有引入前端 UI 框架，也没有使用 CSS 框架。当前页面只需要静态内容和少量布局，原生 Astro 和 CSS 已经足够。

## 结构固定

固定目录结构可以让后续维护更轻松：

- 页面放在 `src/pages`
- 布局放在 `src/layouts`
- 内容放在 `src/content/posts`
- 全局样式放在 `src/styles/global.css`

当文件越来越多时，这种规则比临时决定更可靠。

## 命名清楚

文件名和变量名尽量直接表达用途。比如 `BaseLayout.astro` 负责全站外壳，`PostLayout.astro` 负责文章页面，`SITE_TITLE` 保存站点标题。

清楚的命名不会让项目显得高级，但能减少很多回头阅读时的成本。
